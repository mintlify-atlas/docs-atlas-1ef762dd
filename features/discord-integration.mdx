### Implementation

The widget uses Discord's official embed API:

<CodeGroup>
```jsx Widget component
const Widget = styled.div`
  z-index: 10;
  position: fixed;
  cursor: pointer;
  bottom: 32px;
  right: 48px;
  transition: all 0.3s;
  height: ${({ open }) => (open ? '50%' : '74px')};
  display: inline-block;

  @media (max-width: 600px) {
    right: 24px;
    width: calc(100% - 48px);
  }
`;

<Widget open={openWidget}>
  <Click onClick={() => setOpenWidget(!openWidget)} />
  <iframe
    src="https://discord.com/widget?id=769610692297752586&theme=dark"
    width="100%"
    height="100%"
    allowtransparency="true"
    frameBorder="0"
    sandbox="allow-popups allow-popups-to-escape-sandbox allow-same-origin allow-scripts"
  ></iframe>
</Widget>
```

```jsx State management
const [openWidget, setOpenWidget] = useState(false);

// Toggle widget open/closed
const toggleWidget = () => setOpenWidget(!openWidget);
```
</CodeGroup>

The widget is:
- Fixed to the bottom-right corner of the viewport
- Expandable from 74px to 50% screen height
- Fully responsive on mobile devices
- Styled with dark theme to match the platform aesthetic

<Note>
  The widget requires the Discord server ID (769610692297752586) and must have the widget feature enabled in server settings.
</Note>

## Discord username attribution

When submitting videos to the platform, users provide their Discord username for attribution and communication:

### Submission form

```jsx Discord username field
<Input
  type="text"
  placeholder="Discord Username"
  value={username}
  onChange={(e) => setUsername(e.target.value)}
/>
```

### Stored with submission

Usernames are stored in two locations:

1. **Firebase Storage metadata**: Attached to the video file itself
```jsx
const uploadTask = submissionsRef.put(video, {
  customMetadata: { username, title },
});
```

2. **Firestore document**: Stored with the submission record for easy querying
```jsx
firestore.collection('youtubeSubmissions').add({
  title,
  storageId,
  description,
  tags: chosenTags,
  video: downloadURL,
  createdAt: firebase.firestore.FieldValue.serverTimestamp(),
});
```

<Note>
  While the username is stored in metadata, it's not included in the Firestore document in the current implementation. Consider adding it if you need to query by username.
</Note>

## Widget configuration

To set up the Discord widget for your own server:

<Steps>
  <Step title="Enable widget in Discord">
    Go to Server Settings > Widget and enable the widget feature
  </Step>
  
  <Step title="Get server ID">
    Copy your Discord server ID from the widget settings page
  </Step>
  
  <Step title="Update platform code">
    Replace the server ID in the iframe src:
    
    ```jsx
    <iframe
      src="https://discord.com/widget?id=YOUR_SERVER_ID&theme=dark"
      // ... other props
    ></iframe>
    ```
  </Step>
  
  <Step title="Customize appearance">
    Adjust widget size, position, and styling to match your design
  </Step>
</Steps>

<Warning>
  Make sure your Discord server has the widget feature enabled in server settings, or the embed will not work.
</Warning>

## Responsive behavior

The Discord widget adapts to different screen sizes:

```jsx Responsive styling
const Widget = styled.div`
  position: fixed;
  bottom: 32px;

  height: ${({ open }) => (open ? '50%' : '74px')};
  display: inline-block;

  @media (max-width: 600px) {
    right: 24px;
    width: calc(100% - 48px);
  }
`;
```

**Desktop**:
- Fixed width based on content
- Positioned in bottom-right corner
- Expands to 50% screen height when open

**Mobile**:
- Full width minus padding
- Maintains bottom positioning
- Smaller collapsed height for better mobile UX

## Security considerations

The widget iframe includes security attributes:

```jsx Security settings
<iframe
  src="https://discord.com/widget?id=769610692297752586&theme=dark"
  allowtransparency="true"
  frameBorder="0"
  sandbox="allow-popups allow-popups-to-escape-sandbox allow-same-origin allow-scripts"
></iframe>
```

The sandbox attribute:
- `allow-popups`: Allows joining the server in a new tab
- `allow-popups-to-escape-sandbox`: Allows Discord login flow
- `allow-same-origin`: Required for widget functionality
- `allow-scripts`: Enables interactive features

<Warning>
  Do not remove sandbox attributes without understanding the security implications. These settings balance functionality with security.
</Warning>

## Customization options

You can customize the Discord widget appearance:

### Theme

```jsx
// Dark theme (current)
src="https://discord.com/widget?id=SERVER_ID&theme=dark"

// Light theme
src="https://discord.com/widget?id=SERVER_ID&theme=light"
```

### Size

Adjust the widget dimensions in the styled component:

```jsx
const Widget = styled.div`
  height: ${({ open }) => (open ? '50%' : '74px')}; // Adjust these values
  // ... other styles
`;
```

### Position

Change the widget position:

```jsx
const Widget = styled.div`
  bottom: 32px;  // Distance from bottom
  right: 48px;   // Distance from right
  // Change to left, top as needed
`;
```

<Tip>
  Test widget customizations on both desktop and mobile devices to ensure a consistent experience across screen sizes.
</Tip>
