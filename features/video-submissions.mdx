The video submission feature allows community members to contribute their own technical content to the BLACC YouTube channel. Submit tutorials, coding sessions, or technical talks to share your knowledge with the community.

## Overview

The upload page at `/upload` provides a streamlined workflow for submitting video content. Videos are uploaded to Firebase Storage and queued for review before being published to the official BLACC YouTube channel.

Follow these steps to submit your video to the BLACC YouTube channel:

## Upload process

The platform uses Firebase Storage for video uploads with real-time progress tracking:

<CodeGroup>
```jsx Upload handler
const uploadSubmission = async () => {
  if (validateInputs()) {
    uploadVideoToStorage({
      video,
      username,
      title,
      description,
      chosenTags,
      updateProgress: setProgress,
      onComplete: clearData,
    });
  }
};
```

```jsx Firebase upload
export const uploadVideoToStorage = ({
  video,
  title,
  username,
  description,
  chosenTags,
  updateProgress,
  onComplete,
}) => {
  const storageId = uuidv4();
  const submissionsRef = storage.ref(`youtubeSubmissions/${storageId}`);
  const uploadTask = submissionsRef.put(video, {
    customMetadata: { username, title },
  });

  uploadTask.on(
    'state_changed',
    (snapshot) => {
      const progress =
        (snapshot.bytesTransferred / snapshot.totalBytes) * 100;
      updateProgress(Math.round(progress));
    },
    (err) => {
      throw new Error(`uploadVideoToStorage: ${err.toString()}`);
    },
    () => {
      uploadTask.snapshot.ref.getDownloadURL().then((downloadURL) => {
        firestore
          .collection('youtubeSubmissions')
          .add({
            title,
            storageId,
            description,
            tags: chosenTags,
            video: downloadURL,
            createdAt: firebase.firestore.FieldValue.serverTimestamp(),
          })
          .then(() => setTimeout(() => onComplete(), 1000));
      });
    }
  );
};
```
</CodeGroup>

The upload process:
1. Generates a unique ID for the video using UUID
2. Uploads the video file to Firebase Storage with metadata
3. Tracks upload progress and updates the UI in real-time
4. Creates a Firestore document with video metadata and download URL
5. Shows success confirmation when complete

<Note>
  Videos are stored with a unique identifier to prevent filename conflicts and ensure proper organization.
</Note>

## Progress indicator

A visual progress bar shows upload status:

```jsx Progress UI
<CTAWrapper onClick={onSubmit}>

  <Progress progress={progress}>
    <ProgressWrapper show={progress > 99}>
      <Icon className="material-icons">check_circle</Icon>
      <Message>UPLOAD COMPLETE</Message>
    </ProgressWrapper>
  </Progress>

```

The progress bar:
- Fills from left to right as upload progresses (0-100%)
- Changes color to green when complete
- Displays a checkmark and "UPLOAD COMPLETE" message at 100%

<Tip>
  The upload continues in the background even if you switch tabs, but avoid closing the browser until you see the success message.
</Tip>

## Form validation

The platform validates all required fields before allowing submission:

```jsx Validation logic
const validateInputs = () => {
  if (!chosenTags.length) setError('You must add at least one tag');
  if (!description) setError('You must add a description for the video');
  if (!title) setError('You must add a video title');
  if (!username) setError('You must add your Discord username');
  if (!video) setError('You must upload a video');
  setTimeout(() => setError(''), 5000);
  return video && username && title && description && chosenTags.length;
};
```

Validation checks:
- Video file is selected
- Discord username is provided
- Video title is provided
- Description is provided
- At least one tag is selected

<Warning>
  All fields are required. You'll see an error toast message if any required field is missing.
</Warning>

## Error handling

The platform provides user-friendly error messages:

```jsx Toast notification
<Toast error={error} show={showSuccess} isError={error} />
```

Error messages appear for:
- Missing required fields
- Invalid file types (non-video files)
- Upload failures

Success messages appear when:
- Upload completes successfully
- Form is cleared and ready for new submission

  <Step title="Form reset">

      setVideo(null);

  </Step>

  </Step>
